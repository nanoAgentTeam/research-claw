## 前言

  本手册详细介绍 Open Research Claw Web 控制台的所有功能和操作步骤。手册按照"目标 → 操作步骤 →完成"的形式组织,帮助您快速上手。

---

## 一、启动与访问

  目标:启动 WebUI 并访问控制台

  操作步骤:

1. 打开终端,进入项目目录
2. 执行启动命令:
   python cli/main.py gateway --port 18790
3. 等待启动完成,看到类似以下输出:
   INFO:     Started server process
   INFO:     Uvicorn running on http://127.0.0.1:18790
4. 打开浏览器,访问: http://127.0.0.1:18790/ui
5. 页面加载后,您将看到 Open Research Claw 控制台界面

  完成标志:

- 左侧显示蓝色机器人图标和 "Open Research Claw" 标题
- 左下角显示 "Gateway Status: Connected" (绿色指示灯)
- 主界面显示"控制中心"页面

## 二、首次配置

### 目标 1:配置第一个 LLM 模型

  前提: 您需要准备好 API Key (如 OpenAI、Anthropic、Gemini 等)

  操作步骤:

1. 在左侧导航栏点击 "模型管理" (芯片图标)
2. 页面顶部点击 "+ 添加模型实例" 按钮
3. 填写表单:
   - 实例 ID: 输入唯一标识,如 gpt4
   - 模型名称: 输入要调用的模型名,如 gpt-4-turbo
   - API Base: 输入 API 地址,如 https://api.openai.com/v1
   - API Key: 粘贴您的 API 密钥
   - Provider 类型: 从下拉菜单选择 openai（如果是anthropic协议就选anthropic，url不带v1）
4. 点击 "添加" 按钮
5. 新模型卡片出现在列表中
6. 点击该卡片右上角的 "测试连接" 按钮
7. 等待测试结果,成功会显示绿色提示

  完成标志:

- 模型卡片显示在列表中
- 测试连接成功
- 该模型自动成为"激活模型"(卡片有高亮边框)

---

### 目标 2:配置通讯账号(可选)

  说明: 如果您想通过飞书、Telegram 等与 Agent 交互,需要配置通讯账号

  配置 Telegram Bot

  前提: 已通过 @BotFather 创建 Bot 并获得 Token

  操作步骤:

1. 点击左侧 "通讯账号" (对话气泡图标)
2. 点击 "+ 添加账号" 按钮
3. 在弹出的平台选择中,点击 "Telegram" 卡片
4. 填写表单:
   - Token: 粘贴从 @BotFather 获得的 Bot Token
5. 点击 "添加" 按钮
6. 新账号卡片出现,点击 "测试连接" 按钮验证
7. 测试成功后,点击卡片右上角的开关,确保已启用

  完成标志:

- Telegram 账号卡片显示在列表中
- 测试连接成功
- 开关处于"启用"状态(蓝色)

  配置飞书应用

  前提: 已创建飞书企业自建应用,获得 App ID 和 App Secret

  操作步骤:

1. 在"通讯账号"页面,点击 "+ 添加账号"
2. 选择 "飞书" 卡片
3. 填写:
   - App ID: 输入飞书应用的 App ID
   - App Secret: 输入飞书应用的 App Secret
4. 点击 "添加"
5. 测试连接并启用

  完成标志: 飞书账号卡片显示且测试成功

## 四、推送管理

### 目标:配置推送订阅,接收自动化任务通知

  说明: 推送订阅用于接收定时任务的执行结果通知

#### 配置 Telegram 推送

  前提: 已有 Telegram Bot Token 和 Chat ID

  操作步骤:

1. 点击左侧 "推送管理" (铃铛图标)
2. 在 "推送订阅" 区域,点击 "+ 添加订阅" 按钮
3. 填写表单:
   - 渠道类型: 选择 Telegram
   - Bot Token: 输入您的 Bot Token
   - Chat ID: 输入接收消息的 Chat ID
   - 备注: (可选) 输入描述,如 "我的 Telegram"
   - 启用: 确保开关打开
4. 点击 "保存" 按钮
5. 新订阅出现在列表中
6. 点击该订阅行的 "测试" 按钮
7. 检查 Telegram 是否收到测试消息

  完成标志:

- 订阅列表中显示新订阅
- Telegram 收到测试消息 "这是一条测试推送"

---

#### 配置邮件推送

  前提: 需要 SMTP 邮箱账号

  操作步骤 - 第一步:创建 SMTP 配置

1. 在"推送管理"页面,找到 "SMTP 邮件配置" 区域
2. 点击 "+ 添加 SMTP 配置" 按钮
3. 选择预设或自定义:
   - 如使用 QQ 邮箱,点击 "QQ 邮箱" 预设
   - 如使用 Gmail,点击 "Gmail" 预设
   - 或选择 "自定义"
4. 填写表单:
   - 配置名称: 输入名称,如 "我的QQ邮箱"
   - SMTP 主机: (预设已填) 如 smtp.qq.com
   - 端口: (预设已填) 如 587
   - 用户名: 输入邮箱地址
   - 密码: 输入邮箱密码或授权码
   - 发件人邮箱: 输入发件人邮箱
   - 发件人名称: (可选) 输入显示名称
   - 启用 TLS: 确保开关打开
5. 点击 "保存" 按钮
6. 点击该配置行的 "测试" 按钮
7. 输入测试收件人邮箱,点击发送
8. 检查邮箱是否收到测试邮件

  操作步骤 - 第二步:创建邮件订阅

1. 在 "邮件订阅目标" 区域,点击 "+ 添加邮件订阅"
2. 填写:
   - 收件人邮箱: 输入接收通知的邮箱
   - SMTP 配置: 从下拉菜单选择刚创建的配置
   - 备注: (可选) 输入描述
   - 启用: 确保开关打开
3. 点击 "保存"
4. 点击 "测试" 按钮验证

  完成标志:

- SMTP 配置列表中显示新配置
- 邮件订阅列表中显示新订阅
- 收件邮箱收到测试邮件

## 五、自动化任务

### 目标:创建定时任务

  场景: 每天早上 9 点自动检查论文更新

  操作步骤:

1. 点击左侧 "自动化任务" (时钟图标)
2. 在页面顶部的 "选择项目" 下拉菜单中选择目标项目
   - 如果没有项目,点击 "刷新项目" 按钮
3. 点击 "+ 创建任务" 按钮
4. 填写任务表单:
   - 任务 ID: 输入唯一标识,如 daily_check
   - 任务名称: 输入描述性名称,如 "每日论文检查"
   - 任务类型: 选择 normal
   - 定时规则:
   - 选择 "每天"
   - 时间输入 09:00
     - 时区: 选择您的时区,如 Asia/Shanghai
     - Prompt: 输入任务指令,如:
       检查最近的论文更新,总结重要变化
     - 启用: 确保开关打开
5. 点击 "创建" 按钮
6. 新任务出现在任务列表中

  完成标志:

- 任务列表显示新任务
- 任务状态为"已启用"(绿色开关)
- 显示定时规则 "每天 09:00"

---

### 目标:手动运行任务

  操作步骤:

1. 在任务列表中找到要运行的任务
2. 点击该任务行右侧的 "立即运行" 按钮
3. 等待任务执行
4. 任务状态变为"运行中"(黄色标签)
5. 执行完成后,状态变为"成功"(绿色)或"失败"(红色)

  完成标志:

- 任务执行完成
- 运行历史中出现新记录
- 如配置了推送,收到执行结果通知

---

### 目标:查看任务运行历史

  操作步骤:

1. 在"自动化任务"页面,向下滚动到 "运行历史" 区域
2. 查看历史记录列表,包含:
   - 任务 ID
   - 触发方式(手动/定时)
   - 状态(成功/失败/运行中)
   - 开始时间
   - 结束时间
   - 执行时长
3. 点击某条记录行的 "查看详情" 按钮
4. 展开的详情区域显示完整的执行内容和输出

  完成标志:

- 可以看到所有历史运行记录
- 可以查看每次运行的详细内容

---

### 目标:编辑现有任务

  操作步骤:

1. 在任务列表中找到要编辑的任务
2. 点击该任务行右侧的 "编辑" 按钮
3. 任务行下方展开编辑表单
4. 修改需要更改的字段(如 Prompt、定时规则等)
5. 点击 "保存" 按钮
6. 编辑表单收起,任务更新完成

  完成标志:

- 任务列表中显示更新后的信息
- 页面显示 "任务已更新" 提示

---

### 目标:删除任务

  操作步骤:

1. 找到要删除的任务
2. 点击任务行右侧的 "删除" 按钮
3. 在确认对话框中点击 "确定"
4. 任务从列表中消失

  完成标志:

- 任务不再显示在列表中
- 显示 "任务已删除" 提示

---

### 目标:为项目关联推送订阅

  说明: 让特定项目的任务结果发送到指定的推送渠道

  操作步骤:

1. 在"自动化任务"页面,选择目标项目
2. 向下滚动到 "项目推送订阅" 区域
3. 点击 "关联订阅" 按钮
4. 在弹出的列表中,勾选要关联的全局订阅
5. 点击 "保存" 按钮
6. 关联的订阅显示在项目订阅列表中

  完成标志:

- 项目订阅列表显示已关联的订阅
- 该项目的任务执行后,会向关联的订阅发送通知

## 九、故障排查

### 问题:Gateway 连接失败

  现象: 页面显示 "Gateway 已关闭" 覆盖层

  解决步骤:

1. 检查终端中 Gateway 进程是否还在运行
2. 如果进程已停止,重新执行:
   python cli/main.py gateway --port 18790
3. 等待 Gateway 启动完成
4. 页面会自动检测并恢复连接
5. 覆盖层消失,恢复正常使用

  完成标志:

- 左下角显示 "Connected" (绿色)
- 可以正常操作所有功能

---

### 问题:模型测试连接失败

  可能原因:

- API Key 错误
- API Base 地址错误
- 网络连接问题
- Provider 类型选择错误

  解决步骤:

1. 点击模型卡片的 "编辑" 按钮
2. 检查并更正:
   - API Key 是否完整正确
   - API Base 是否包含正确的协议(https://)
   - Provider 类型是否匹配
3. 点击 "保存"
4. 再次点击 "测试连接"
5. 查看错误提示,根据提示调整

  完成标志:

- 测试连接成功
- 显示绿色成功提示

---

### 问题:推送测试失败

  针对 Telegram:

- 检查 Bot Token 是否正确
- 检查 Chat ID 是否正确
- 确认 Bot 已被添加到对应的聊天中

  针对邮件:
- 检查 SMTP 配置是否正确
- 检查用户名和密码
- 确认 TLS 设置是否正确
- 检查端口号(通常 587 用于 TLS,465 用于 SSL)

  解决步骤:

1. 点击订阅的 "编辑" 按钮
2. 检查并更正所有字段
3. 点击 "保存"
4. 再次点击 "测试"
5. 查看错误消息,根据提示调整

  完成标志:

- 测试成功
- 收到测试消息

---

### 问题:任务执行失败

  查看失败原因:

1. 在"自动化任务"页面,找到 "运行历史" 区域
2. 找到失败的运行记录(红色"失败"标签)
3. 点击 "查看详情" 按钮
4. 查看详细的错误信息和日志
5. 根据错误信息调整任务配置或 Prompt

  常见问题:

- Prompt 不清晰或有歧义
- 模型配置错误
- 项目路径问题
- 权限问题

  完成标志:
- 找到失败原因
- 调整后任务可以成功执行

---

## 十、最佳实践

### 配置管理

1. 定期备份配置: 在做重大更改前,先在"控制中心"创建配置备份
2. 测试后保存: 添加模型或账号后,先测试连接,确认成功再保存
3. 使用描述性名称: 为模型实例、任务、订阅使用清晰的名称和备注

### 任务管理

1. 从简单开始: 先创建简单的测试任务,确认流程正常后再创建复杂任务
2. 合理设置定时: 避免设置过于频繁的定时任务,以免消耗过多资源
3. 监控运行历史: 定期查看运行历史,及时发现和解决问题
4. 使用推送通知: 为重要项目配置推送订阅,及时了解任务执行情况

### 安全建议

1. 保护敏感信息: 不要在公共场合展示包含 API Key 的屏幕
2. 定期更换密钥: 定期更换 API Key 和密码
3. 限制访问: Gateway 默认只监听 127.0.0.1,不要随意修改为 0.0.0.0
4. 备份重要数据: 定期备份配置和自动化任务设置
