以下内容与 `README/guide/push_subscription_channels.md` 同步，便于在当前页面直接配置。

1) Telegram
字段：`params.bot_token`、`params.chat_id`
获取：联系 `@BotFather` 创建机器人拿 `bot_token`；给机器人发消息后访问 `https://api.telegram.org/bot<bot_token>/getUpdates` 获取 `chat.id`。
2) Feishu（飞书自定义机器人）
字段：`params.webhook_url`
获取：在群里添加自定义机器人，直接粘贴完整 Webhook URL，形如 `https://open.feishu.cn/open-apis/bot/v2/hook/xxxx`。
3) QQ Push（qmsg）
字段：`params.token`
获取：在 `https://qmsg.zendee.cn/` 注册获取key。
按照文档指示（https://qmsg.zendee.cn/docs/start/）：添加Qmsg酱为好友等；Webhook 通常为 `https://qmsg.zendee.cn/send/<token>`。
4) 微信（企业微信群机器人 / WeCom Bot）
字段：`params.key`
获取：在企业微信群添加机器人，Webhook 形如 `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=<key>`。普通微信个人号不支持该官方 webhook。
5) Server酱（方糖）
字段：`params.sendkey`
获取：在 `https://sct.ftqq.com/` 登录后创建并复制 `SendKey`（通常以 `SCT` 开头）。
6) custom（自定义 Apprise URL）
字段：`apprise_url`
直接填写完整 URL（如 `mailto://...`、`json://...`）。服务格式可参考 `https://appriseit.com/services/`。
7) Email（SMTP 邮件）
字段：`params.email`、`params.profile_id`（可选）
步骤：1）在上方新增 SMTP 配置（选择服务商预设 → 填“邮箱账号/授权码/发件邮箱/发件人名” → 测试）。2）在“SMTP 邮件配置（快速模式）”中的“邮件订阅目标（收件人）”里填写 `params.email`，可选指定 `params.profile_id`。
说明：邮箱账号通常是登录用的地址，发件邮箱是邮件“From”显示的地址。大多数情况下二者相同；有些服务允许“代发”或别名地址所以分开。
快速排查：先点“测试”单条，再点“测试全部启用订阅”；若失败，优先检查字段名是否对应渠道要求、值里是否包含多余空格。
