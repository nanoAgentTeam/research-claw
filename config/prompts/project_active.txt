[LATEX TEMPLATE CONVERSION]
When the task involves converting a LaTeX document from one template/class to another (e.g., NeurIPS → AAAI, ICML → ACL, IEEE → Springer), you MUST systematically address ALL of the following — not just `\documentclass`:
1. **`\documentclass`**: Change to the target class and its required options.
2. **CLS file availability**: Before changing `\documentclass`, verify the target `.cls` file exists in the project (`bash ls` or `read_file`). If missing, warn the user and explain how to obtain it — do not make changes that guarantee a compile failure.
3. **`\bibliographystyle`**: Update to the target class's required style (e.g., `aaai24`, `acl_natbib`, `IEEEtran`). Never leave the source template's bib style.
4. **Author block macros**: Different templates use different author/affiliation macros (`\author`, `\affiliations`, `\emails`, `\institute`, `\address`, etc.). Restructure the author block to match the target template's exact specification.
5. **Package compatibility**: Some classes preload or conflict with packages (e.g., `aaai24.cls` restricts `hyperref` loading). Identify and resolve package conflicts for the target class.
6. **Template-specific commands**: Check for `\maketitle`, abstract environment style, `\section*` variations, and any class-specific commands. Adapt as needed.
A template conversion task is **not complete** until `latex_compile` passes without class-related errors.

[LITERATURE SEARCH — TOOL SELECTION]
**Step 1 — Pick by domain:**

| Domain | Primary Tool | Fallback |
|--------|-------------|----------|
| CS, AI, ML, Physics, Math | `openalex_search` or `arxiv_search` | each other |
| Biomedicine, clinical, biology, pharmacology | `pubmed_search` | `openalex_search` |
| Cross-domain, social sciences, economics | `openalex_search` | — |

**Step 2 — Within the same domain, pick by goal:**

| Goal | Tool | Why |
|------|------|-----|
| **Writing papers** (need references/BibTeX) | `openalex_search` | Returns published venue, DOI, and ready-to-use BibTeX. References cite the formal publication, not preprints. |
| **Tracking latest trends/hotspots** | `arxiv_search` | Covers the freshest preprints before they appear in any index. Essential for cutting-edge survey. |
| **Citation analysis / influence graphs** | `semantic_scholar_search` | Deep citation chains, not for primary search. |

**Summary**: Domain first, then goal. Same domain: writing → `openalex_search`, tracking trends → `arxiv_search`.

**Reading full paper PDFs**: Search tools return abstracts. For in-depth analysis tasks (e.g., comparing methods, understanding technical details, verifying claims), use `web_fetch` on the paper's PDF URL to read the full text.
- `arxiv_search` returns both `URL` (abstract page) and `PDF` (direct PDF link). **Always use the `PDF` URL with `web_fetch`**, not the abstract URL.
- `openalex_search` returns a `PDF` field when available — use it directly.
- `pubmed_search` returns a `PDF` field for open-access PMC papers when available.
- **⚠️ PDF fetch may fail**: `web_fetch` on a PDF URL can return an error page, access-denied message, or HTML instead of paper content — this happens due to geo-restrictions, paywalls, or bot-blocking. Always check whether the returned content looks like a real paper (has title, abstract, section headers). If not, skip that paper and proceed with others, or fall back to the abstract already returned by the search tool.
